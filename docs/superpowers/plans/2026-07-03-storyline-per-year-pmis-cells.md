# Storyline Per-Year PMIS Cells Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Color each Storyline window bar as per-year PMIS condition cells instead of one mean-condition color.

**Architecture:** The data generator (`step17_storyline_data.py`) emits a per-year score array `yv` alongside the existing per-window mean `v`. The renderer (`storyline.js`) gains a `pmisCategoryColor()` helper and, in **condition** color mode, subdivides each window bar's x-extent into N equal hard-edged cells (N = `yv.length`), each colored by that year's PMIS category. Cohort mode, connectors, and the hover-highlight bar are untouched. If `yv` is absent, the renderer falls back to the current single-color bar.

**Tech Stack:** Python 3 + numpy (data generator), vanilla ES2017 + WebGL + Canvas-2D + d3 (renderer), `unittest` (Python tests).

## Global Constraints

- PMIS palette is authoritative and copied verbatim from `heatmap.js:15-37`:
  - `score >= 90` → Very Good → `rgb(21,128,61)`
  - `score >= 70` → Good → `rgb(34,197,94)`
  - `score >= 50` → Fair → `rgb(234,179,8)`
  - `score >= 35` → Poor → `rgb(249,115,22)`
  - `score >= 1`  → Very Poor → `rgb(239,68,68)`
  - `score < 1`   → Invalid → `rgb(200,200,200)`
  - `null` / `undefined` / `NaN` → No data → `#999999`
- Per-year cells apply to **condition** color mode ONLY. Cohort mode is untouched.
- Connectors between windows are untouched (still colored by mean `v`).
- The hover-highlight bar (`drawHighlighted`) is untouched (stays single mean-`v` color; no per-year hover).
- Keep the existing `v` field in the data — connectors, cohort mode, and the fallback path use it.
- Windows slide by 1 year (W=5, 25 windows, 1996–2024); per-year cells intentionally repeat across overlapping windows.
- Only `step17_storyline_data.py` is edited on the data side. `export_storyline_data.py` writes `storyline_data.json`, which no page loads — do NOT touch it.
- `storyline_data_units.json` (loaded by `index_units.html`) is generated by a different script and is NOT regenerated here; it must keep working via the `yv`-absent fallback.
- Run all Python commands from the repo root `E:\GeoChron`. On this machine, `graphify` lives at `~/.local/bin` (add to PATH for the graph-update step).

```python
"""Integration test: the storyline data generator emits a per-year `yv`
array on every window entry, consistent with the mean `v`."""
import json, os, subprocess, sys, unittest

ROOT = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))


def regenerate(rule):
    subprocess.run([sys.executable, "step17_storyline_data.py", rule],
                   cwd=ROOT, check=True)
    with open(os.path.join(ROOT, f"storyline_data_{rule}.json"), encoding="utf-8") as f:
        return json.load(f)


class TestYv(unittest.TestCase):
    @classmethod
    def setUpClass(cls):
        cls.data = regenerate("hwcounty")

    def _window_len(self, k):
        w = self.data["windows"][k]
        return w["end"] - w["start"] + 1

    def test_every_window_entry_has_yv_of_window_length(self):
        checked = 0
        for road in self.data["roads"]:
            for seg in road["segments"]:
                for w in seg["win"]:
                    self.assertIn("yv", w)
                    self.assertEqual(len(w["yv"]), self._window_len(w["k"]))
                    checked += 1
        self.assertGreater(checked, 0)

    def test_mean_of_nonnull_yv_matches_v(self):
        for road in self.data["roads"]:
            for seg in road["segments"]:
                for w in seg["win"]:
                    vals = [x for x in w["yv"] if x is not None]
                    if not vals:
                        self.assertIsNone(w["v"])
                        continue
                    self.assertAlmostEqual(round(sum(vals) / len(vals), 1),
                                           w["v"], delta=0.15)


if __name__ == "__main__":
    unittest.main()
```

- [ ] **Step 1: Add the `pmisCategoryColor` helper**

In `storyline.js`, directly after the `conditionColor` function (ends at L138, before the `CATEGORICAL` const at L140):

```javascript
  // Per-year PMIS condition palette, copied verbatim from heatmap.js so the
  // Storyline year-cells match the heatmap view. Used only in condition mode.
  function pmisCategoryColor(score) {
    if (score === null || score === undefined || isNaN(score)) return "#999999";
    if (score >= 90) return "rgb(21,128,61)";   // Very Good
    if (score >= 70) return "rgb(34,197,94)";   // Good
    if (score >= 50) return "rgb(234,179,8)";   // Fair
    if (score >= 35) return "rgb(249,115,22)";  // Poor
    if (score < 1)   return "rgb(200,200,200)"; // Invalid
    return "rgb(239,68,68)";                     // Very Poor
  }
```
